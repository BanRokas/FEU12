Libraries ir Frameworks

CSS Frameworks/Libraries - kažkieno kito parašytas CSS kodas, kurį tu gali naudoti savo projektuose. Norint naudotis framework'u reikia jį prijungti prie savo projekto naudojant CDN. Dauguma CSS framework'ų veikia tokiu principu - prijungus prie savo projekto gali naudoti jų viduje aprašytas klases - priskirinėti HTML elementams framework'o klases, o pats gali net nerašyti jokio CSS. Yra Classless CSS framework'ų. Juos naudojant, nereikia suteikti HTML elementams klasių. Viskas yra aprašyta naudojant tag'ų/nth/attributų... selektorius.

Pros

  • Pagreitina/Palengvina darbą
  • Nereikia/labai mažai rašyti CSS
  • Lengva naudotis
  • Įdiegtas responsiveness
  • ...

Cons

  • Daug perteklinio/nenaudojamo kodo
  • Reikia skirti laiko išmokti
  • Pasimiršta CSS
  • Nešvarus HTML dėl klasių pertekliaus
  • Neunikalus dizainas
  • Prastenis google paieškos sistemos (SEO) įvertinimas

Preprocessors

CSS with superpowers

Suteikia galimybę rašyti CSS'ą naujoviškesniu būdu, kur galima lengvai kurti kintamuosius, nest'inti selektorius, rašyti: sąlygas, funkcijas, mix'inus ir t.t.

Naudojant betkurį CSS preprocesorių, reikia parašyta kodą konvertuoti į paprastą CSS'ą, kurį supras svetainė.

Preprocesorių sąrašas

  • SASS/SCSS
  • LESS
  • Stylus
  • PostCSS
  • ...

Units

  • px
  • % - tėvinio elemento dydį
  • vh - viewport height - matomos svetainės dalies aukštis (100 - visas aukštis)
  • vw - viewport width - matomos svetainės dalies plotis (100 - visas aukštis)
  • rem - root em - 16px - šakninius nustatymus
  • em - 16px - tėvinio elemento nustatymus
