API

Application programming interface - jūsų užklausa iš aplikacijos (front-end) išsiunčiama į serverį (back-end) ir jeigu viskas - grąžinami duomenys pagal jūsų užklausą, jeigu kažkas negerai - grąžinama atitinkama error'o žinutė.

JSON

JSON - Java Script Object Notation yra skirtas paversti duomenis į tokį formatą, kuris galėtų būti siunčiamas su API.
JSON formatas nuo JS Object formato skiriasi tuo, kad raktiniai žodžiai turi būti kabutėse, o pačios kabutės visada dvigubos.

Tam, kad paversti JS sintaksę į JSON formatą arba atvirkščiai, naudojamos dvi JSON komandos:
JSON.stringify(JS syntax) - iš JS sintaksės paverčia ir grąžiną JSON sintaksės string'ą.
JSON.parse(JSON syntax) - iš JSON sintaksės paverčia ir grąžina JS sintaksę (kaip objektą ar masyvą).

Fetch

Funkcija, kurios pagalba mes galime siūsti užklausas į serverį ir gauti kažkokius atsakymus.

Fetch vykdymo eiga:

  1. Fetch'inant (siūsčiant užklausą) nurodomas URL, galima nurodyti ir papildomas opcijas. Tarp tų opcijų yra metodai:
    • GET - gauti info
    • POST - pateikti info
    • DELETE - trinti info
    • PATCH - redaguoti info
    • PUT - redaguoti info
  2. Apdoroti gautą info (vykdosi kai nėra error)
  3. Naudoti gautą info (vykdosi kai nėra error)
  4. Pagaunam (catch) ir kažką atliekam su tuo error (vykdosi tik erorr atveju)
  5. Galiausiai (finally) dar kažką atliekam (vykdosi betkokiu atveju)
