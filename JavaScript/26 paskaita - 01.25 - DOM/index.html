HTML formų validavimas

Formų validavimas - tikrinimas ar formos įvesčių laukai teisingai supildyti.
Formų validavimas yra reikalingas tam, kad nebūtu leidžiama pateikti formos, jeigu kažkurie laukai neteisingai supildyti arba išvis nesupildyti.

  • required - įvesties laukas privalomas
  • minlength - minimalus kiekis simbolių - text/password/url/email
  • maxlength - maksimalus kiekis simbolių - text/password/url/email
  • min - mažiausias galimas skaičius - number/range/date
  • max - didžiausias galimas skaičius - number/range/date
  • step - kokiu žingsniu galima įvesti skaičių - number/range/date
  • pattern - spacifikuota seka (su regex) - password/tel
  • accept - kokio tipo failą/us priima - file
  • multiple - leidimas daugiau įvesčių į vieną - file/email

DOM - Document Object Model

Node

Node'as - HTML elementas JS'o aplinkoje.

Galima HTML pildyti innerHTML'u - rašant HTML kodą JS'o string'e ir šį įklijuoti į HTML dokumentą, kur to string'o viduje esantis HTML kodas bus skaitomas kaip HTML ir atvaizduojamas vartotojui.
Bet šitas būdas turi daug minusų, pradedant nuo saugumo, baigiant implementacija.

Geresnis būdas pildyti HTML'ą su JS'u - kurti HTML elementus (node'us) JS'e ir juos append'inti į kažkurį HTML'o elementą.
Node'ai yra kuriami ir modifikuojami naudojant Node'ų metodus.

Node'ų metodai

  • document.createElement('tagVardas')
  • document.createTextNode('kažkoks tekstas...')
  • kam.append(ką0, ką1, ką2, ...) - į kokį node'ą įterpi kokį/ius node'ą/node'us/tekstus
  • kam.appendChild(ką) - į kokį node'ą įterpi VIENĄ kitą NODE'ą
  • kam.prepend(ką0, ką1, ...) - tas pats kas append, tik prideda į pradžią, o ne galą
  • kam.setAttribute('name', 'value')
  • kam.classList
    • add('vienaKlase', 'kitaKlase', ...)
    • remove('vienaKlase', 'kitaKlase', ...)
    • replace('klaseIsimi', 'klasePridedi')
    • toggle('klase')
